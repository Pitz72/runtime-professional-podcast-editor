# v0.0.1 — "Fondamenta"
**Data di rilascio:** 13 Luglio 2026

---

## 🎯 Obiettivo

Reboot completo del progetto. La revisione totale del codice (Luglio 2026) ha rivelato che l'app era di fatto **una web app che fingeva di essere desktop**: il bridge Electron non era mai usato, il menu nativo non faceva nulla, lo stato era diviso tra due sistemi incompatibili, e circa un terzo del codice era morto o irraggiungibile — inclusi file che rompevano la compilazione e l'intera suite di test.

Questa versione risolve **tutte le criticità gravissime** individuate e riparte con il versioning da 0.0.1.

---

## 🏗️ Cosa è cambiato

### 1. Stato unificato (fix split-brain)
Prima esistevano due applicazioni parallele: uno store Zustand (con undo/redo) usato solo da componenti mai montati, e la UI reale basata su `useState` locali. Ora lo store Zustand è l'**unica fonte di verità**: progetto, selezione, zoom, history, dirty flag, path del progetto.

Conseguenza diretta: **undo/redo funziona davvero**, dal menu nativo (Ctrl+Z / Ctrl+Y), con snapshot strutturali che preservano gli `AudioBuffer` decodificati (un deep-clone JSON li avrebbe distrutti).

### 2. Integrazione Electron reale
- Menu File/Edit → azioni vere via IPC: New, Open, Save, **Save As** (nuovo), Export, Undo, Redo.
- Salvataggio e caricamento progetti con **dialog nativi** e path ricordato (Ctrl+S salva senza richiedere il percorso).
- Import audio con dialog nativo (pulsante "+ Import" nel File Bin) oltre al drag & drop dall'OS.
- **Dirty flag**: chiudere la finestra con modifiche non salvate mostra una conferma nativa.
- Preload riscritto: bridge tipizzato (`electron.d.ts`), eventi menu con unsubscribe, `webUtils.getPathForFile` per risolvere i path dei file trascinati (l'API `File.path` non esiste più in Electron 39).

### 3. Persistenza professionale
Il formato progetto ora salva i **path assoluti** dei file audio (niente più Data URL base64 che gonfiavano il JSON del 33% e raddoppiavano la RAM). Al caricamento i file vengono riletti dal disco e ri-decodificati; i file mancanti vengono segnalati senza corrompere il progetto. Il formato ha uno `schemaVersion` e una validazione completa (clip orfani scartati, volumi clampati, tipi verificati campo per campo).

### 4. Sicurezza
- **Gemini eliminato del tutto**: niente API esterne, niente chiavi, niente costi per gli utenti. La chiave API non è più compilata nel bundle distribuito (prima chiunque poteva estrarla). Restano i 22 preset curati locali.
- IPC filesystem con allowlist: si scrive **solo** su path scelti dall'utente via dialog; letture limitate a estensioni audio/progetto. Tutto I/O asincrono.
- `sandbox: true`, `webSecurity` sempre attivo, popup negati, navigazione esterna bloccata, single-instance lock, CSP senza endpoint remoti.

### 5. Bug audio corretti
- **Catena EQ orfana**: se un preset non aveva il compressore (tutti gli 8 preset musica!), i filtri EQ venivano creati ma nessuna sorgente ci passava dentro — effetti silenziosamente inesistenti. Corretto l'ordine di costruzione della catena.
- **Export onesto**: FLAC/AAC erano WAV rinominati; rimossi. Restano WAV e MP3 reali, salvati tramite dialog nativo.
- Fix del bug di precedenza in `encodeWAV` che scalava male i sample negativi.

### 6. Base tecnica risanata
- `tsc --noEmit` pulito con **`strict: true`** su renderer e main.
- ESLint 9 (flat config) installato e verde — prima lo script `lint` non aveva né config né eslint installato.
- Suite Vitest riscritta: **16 test verdi** (prima: 0, la suite non si caricava per un import rotto).
- Eliminati ~2.500 righe e 13 file di codice morto, tra cui `UndoRedoHistory.tsx` che importava un hook inesistente e rompeva la compilazione.

## ⚠️ Note di compatibilità

- I progetti salvati con le versioni 0.x precedenti (con audio embedded come Data URL) **non ricaricano l'audio**: il nuovo formato referenzia i file per path. Reimportare i file audio e risalvare.
- La funzione "AI Enhancement" non esiste più per scelta di prodotto: nessun costo API a carico degli utenti.
