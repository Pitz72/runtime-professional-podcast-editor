# v0.5.2 — "Il Gran Consolidamento (Hotfix 2)"
**Data di rilascio:** 24 Febbraio 2026

---

## 🎯 Obiettivo
Questo ulteriore micro-hotfix risolve una regressione critica introdotta durante lo sviluppo della v0.5.1 che impediva il corretto drag and drop dell'audio nella timeline.

---

## 🔴 Fix Critici e Miglioramenti

### 1. SyntaxError in Drag and Drop (Timeline)
**Problema:** Trascinare un file dal File Bin alla Timeline falliva e generava un crash `SyntaxError: Unexpected end of JSON input` visibile in console.
**Causa:** Nella patch precedente, nel tentativo di alleggerire il `dataTransfer` da file pesanti (`AudioBuffer`), il tipo MIME standard `application/json` veniva processato in modo anomalo in Electron. In certi scenari di UI restituiva una stringa vuota, su cui `JSON.parse` collassava. Inoltre, era presente un errore di sintassi TypeScript nel restore del blocco `try/catch`.
**Soluzione:**
1. Introdotto un Formato MIME custom per l'app: `application/runtime-audio-file`. Questo assicura che il sistema operativo, Electron o il browser non svuotino il pacchetto dati scambiandolo per una stringa testuale standard.
2. Aggiunto un controllo di sicurezza in `Timeline.tsx` che previene l'esecuzione di `JSON.parse` nel caso il getter restituisca dati assenti (ad esempio se un utente trascina file "estranei" direttamente dal sistema operativo sulla timeline).
3. Ripristinata la corretta struttura sintattica del blocco `catch`.

> [!WARNING]
> **Stato Attuale:** Nonostante i fix applicati in questa patch e la risoluzione degli errori di sintassi in console, il trascinamento fisico dell'audio dal File Bin alla Timeline fallisce ancora o non sortisce l'effetto di drop visivo. La problematica richiederà un'indagine più profonda sul comportamento nativo degli eventi di Drag & Drop tra componenti React complessi o sulle API HTML5 in Electron. Lo sviluppo è in pausa su questo punto.
